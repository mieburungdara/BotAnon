# Perbaikan Schema Tabel Chats: Menggunakan Telegram ID Langsung

## Status
✅ Rencana siap untuk diimplementasikan

## Masalah
Saat ini tabel `chats` menggunakan `user1_id` dan `user2_id` yang mereferensikan `id` (row id internal) dari tabel `users`. Ini menyebabkan:
1. Selalu perlu lookup user 2x hanya untuk mendapatkan telegram_id partner
2. Query menjadi lambat karena perlu JOIN tambahan
3. Logika kode menjadi lebih kompleks
4. Resiko race condition ketika user baru dibuat

Padahal tabel `messages` sudah benar menggunakan `sender_telegram_id` langsung.

## Solusi
Ubah schema tabel `chats` untuk menyimpan `user1_telegram_id` dan `user2_telegram_id` langsung.
